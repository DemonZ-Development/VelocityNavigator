# Marketplace and Brand Assets

Brand artwork and guide headers are generated by `scripts/generate_marketplace_assets.py`. The Java and Bedrock selector images are real in-game screenshots, and the dashboard preview is a real runtime capture. These screenshots are never regenerated by that script.

The same script also writes the compact page headers under `wiki/headers/` so the public guides share the same visual style.

The visual system uses a navy background, off-white type, and one cyan accent. Avoid adding glow, gradients with heavy bloom, stock gaming artwork, or dense paragraphs inside images.

## Listing sources

`modrinth-listing.md` is the canonical marketplace body. Run `scripts/generate_marketplace_assets.py` after editing it; the script refreshes `hangar-listing.md` and `spigot-resource.md` with the same feature copy and platform-specific download wording. `spigot-resource.bbcode` is the separately formatted Spigot resource-page source.

Marketplace bodies keep players on their current marketplace for downloads. They must not link to GitHub releases or use GitHub as a download CTA.

Keep critical compatibility, installation, command, permission, and limitation details as real text. Modrinth requires meaningful image alt text, and Spigot's premium guidelines discourage substantive description text being placed only inside images.
