# Real Velocity + Paper integration stack

This is a local, loopback-only integration environment for VelocityNavigator. It runs the built universal JAR in both of its real modes:

- Velocity proxy: `127.0.0.1:25565`
- Paper backend: `127.0.0.1:25566`

The ignored `test_env/` directory receives downloaded server JARs, worlds, logs, configuration, and generated plugin data. No runtime files belong in Git.

## Requirements

- Windows PowerShell 5.1+
- Maven and Java 21+
- Internet access to PaperMC's downloads service
- Minecraft Java Edition client for the final player-routing and inventory-menu check

The stack intentionally pins Paper `1.21.11` and Velocity `3.5.1`. Paper's current documentation lists Java 21 for Paper 1.20–1.21.11, while Paper 26.1+ needs Java 25; the local environment currently has Java 21. The setup uses PaperMC's documented downloads service with its required identifiable `User-Agent` and verifies SHA-256 checksums. [Paper Java requirements](https://docs.papermc.io/paper/getting-started/) · [Paper downloads service](https://docs.papermc.io/misc/downloads-service/)

## Use

Run these from the repository root:

```powershell
.\scripts\integration\VelocityNavigator-Integration.ps1 -Action Setup
.\scripts\integration\VelocityNavigator-Integration.ps1 -Action Start -AcceptMinecraftEula
.\scripts\integration\VelocityNavigator-Integration.ps1 -Action Status
.\scripts\integration\VelocityNavigator-Integration.ps1 -Action Stop
```

`-AcceptMinecraftEula` is deliberately required before Paper can start. Supplying it confirms that you agree to the Minecraft EULA. Paper requires this agreement to run. [Paper EULA reference](https://docs.papermc.io/paper/reference/vanilla-data-files/)

The generated proxy uses Velocity modern forwarding and a random per-environment forwarding secret. The backend is loopback-bound, and both proxy and backend use test-only offline mode. Never copy this configuration to a public production server; production must also be protected with a firewall. [Velocity forwarding configuration](https://docs.papermc.io/velocity/player-information-forwarding/) · [Velocity security guidance](https://docs.papermc.io/velocity/security/)

## What the smoke test proves

`Status` verifies live TCP listeners and confirms, from actual process logs, that the same release JAR has loaded as both the Velocity proxy plugin and the Paper backend bridge.

## Node-player GUI smoke test

With the stack running, install Mineflayer only inside the ignored test environment and run the GUI test:

```powershell
npm install --prefix test_env/node-player mineflayer@4.37.1
node scripts/integration/NodePlayerGuiTest.js
```

The test logs an offline-mode Java player into Velocity, waits for Paper to open the native Bukkit inventory generated by `/lobby menu`, clicks its lobby item, and requires Velocity's same-server response. This validates the actual proxy-to-backend `OpenMenu` packet and backend-to-proxy selection packet; it is not a simulated protocol test.

To exercise the native party commands with two simultaneous players:

```powershell
node scripts/integration/NodePartySmokeTest.js
```

This covers invite/accept, two-member status, private party chat delivery to both clients, member leave, updated leader status, and disband cleanup against the real Velocity command implementation.

The capacity-queue smoke test needs a deliberately full lobby pool and a separate holding-server name. In the ignored test configuration, give `lobby` a finite capacity of `0`, register `holding` to the Paper fixture address, and set `queue.holding_server = "holding"`. Restart Velocity, then run:

```powershell
node scripts/integration/NodeQueueSmokeTest.js
```

It verifies initial holding-server routing, automatic queue entry, `/queue`, `/queue leave`, and removal from the live queue. Restore the normal lobby capacity and holding-server setting after the test.

For Redis pub/sub integration without installing a system service, start the
scoped RESP fixture on port `16379`:

```powershell
node scripts/integration/NodeRedisTestServer.js 16379
```

The fixture implements only the Redis commands used by this integration test.
`-Action RestartBackend` restarts only Paper while Velocity stays subscribed,
allowing the backend bridge's signed registration to be tested reliably.

For the remaining end-to-end interaction test, connect a Java client to `127.0.0.1:25565`, join the lobby, and run `/lobby`. This validates actual client forwarding, initial routing, and (when configured for inventory mode) the backend plugin-message handshake. The log files are under `test_env/proxy/console.log` and `test_env/backend/console.log`.
